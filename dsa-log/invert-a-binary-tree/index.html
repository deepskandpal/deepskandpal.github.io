    [[Recursion]] [[Binary trees and Binary search trees]]

    Problem

    You are given the root of a binary tree. Your task is to swap the left and right child of every single node in the tree. The result is a tree that is a mirror image of the original.

    Brainstorming

    The idea is , when you start to swap from the root you go down to each node and swap the left and the right leafs till the time you don’t hit a dead end. The best way to do this is through a traversal but which one?

    lets take an example

       A
  / \
 B.  C
/ \ / \
D E F. G

    When we do a swap in this example we will start at the root

    1. at A swap its children B and C
    2. at B swap its children D and E
    3. at C swap its childer F and G

    In this top down approach the one thing is clear we cannot swap the current node before swapping the children node hence it has to be bottom up (we swap the childern first before we swap the parent), and the best traversal to do is post order traversal

    the algorithm

    1. check if root is None is if yes then just return
    2. if not first invert the left tree invertBinaryTree(root.left)
    3. then invert the right tree invertBinaryTree(root.right)
    4. once the children are inverted its not time to invert the parents root.left, root.right = root.right, root.left
    5. Return: Return the node itself, so the caller can maintain the link to the (now inverted) subtree.

    Below is the python code

    
class BinaryTree:
	def __init__(self, val, left=None, right=None):
		self.val = val
		self.left = left
		self.right = right
		

def invertBinaryTree(root:BinaryTree):
	if root is None:
		return
	
	invertBinaryTree(root.left)
	invertBinaryTree(root.right)
	root.left , root.right = root.right , root.left
	
	return root

    Space complexity is O(1) and time complexity is O(h) where h is the height of the binary tree
