    [[Linked List Stacks and Queues a Primer]]

    Problem

    Problem: Reverse a Linked List (The whole thing).

    • Goal: If we have 11 -> 3 -> 5 -> None, we want to end up with 5 -> 3 -> 11 -> None.

    Brainstorming

    The idea is we cannot start changing the the linked list directly otherwise we will break the continuation by losing the information

    for eg in the example the current head 11 points to 3 but in the reversed form it points to None if we make 11|next-> None then we will not know the address of the next pointer of 3

    In order to make sure that does not happen before we proceed with the reverse operation we need to save the next node in a pointer next_node = current.next Now coming to the operation of reverse , we would need to make the current node previous the way it will happen is by default the previous is None because the new head will point to None current.next = previous

    now that we have reversed the node we now need to move the pointers for previous and current

    previous = current current = next_node

    below is the python code

    class Node:
	def __init__(self, data=None , next_node=None)
		self.data = data
		self.next = next_node

"""
11->3->6
"""
node_11 = Node(11)
node_3 = Node(3)
node_6 = Node(6)

node_11.next = node_3
node_3.next = node_6

head = node_11

# Traversing a linked list
current_node = head
previous = None

while current_node:
	next_node = current_node.next # save the next node 
	current_node.next = previous # point the current node next to previous
	previous = current_node # assingn the previous node the current node 
	current_node = next_node

    Variation: Reverse a Sublist

    Problem: Reverse a sublist from position s to f.
    L = 11 -> 3 -> 5 -> 7 -> 2 -> None, s=2, f=4
    Goal: 11 -> 7 -> 5 -> 3 -> 2 -> None

    The Game Plan:

    1. Find the starting gate: We need to get to the node right before the reversal starts. In our example, we need a pointer to 11.
    2. Run the race: Perform the exact reversal logic you just wrote, but only for the sublist portion (3 -> 5 -> 7).
    3. Connect the finish line: After the reversal (7 -> 5 -> 3), we need to connect 11 to 7 and 3 to 2. This is the trickiest part.

    To make Step 1 and 3 easy, we use the Dummy Head pattern. We create a fake node that points to our real head. This way, even if we need to reverse from the very first element (s=1), our “node before” logic still works.

    
class Node:
	def __init__(self, data=None , next_node=None)
		self.data = data
		self.next = next_node
		
def reverse_sublist(L, start, finish)
	dummy_node = Node(0, L)
	
	prev_head = dummy_node
	
	# iterate till the head is just before the start 
	for _ in range(start -1):
		prev_head = prev_head.next
		
	
	# now we will apply our reverse logic from start to finish 
	current_node = prev_head.next
	for _ in range(finish-start):
		next_node = current_node.next
		current_node.next, next_node.next, prev_head.next = next_node.next, prev_head.next, next_node 
		
	

"""
11 -> 3 -> 5 -> 7 -> 2 -> 
"""
node_2 = Node(2, None)
node_7 = Node(7, node_2 )
node_5 = Node(5,node_7 )
node_3 = Node(3,node_5)
node_11 = Node(11, node_3)
