    A Trie (from retrieval) data structure is used to mainly represent words in a sum of its prefix parts.

    A Trie class consists of 3 operations

    1. Insert ( adding a word)
    2. Search (finding if the exact word is in trie or not)
    3. Startswith(prefix) check if the prefix of the trie is present or not

    A Trie node contains 2 pieces of information

    1. Information of the next character example dict
      1. {“a”: node_of_a, “b”: node_of_b}
      2. Is this node the end of a word is_end_of_word = True/False

    A Trie node would look like this

    class TrieNode:
	def __init__(self):
		self.children = {}
		self.is_end_of_word = False

"""
trie tree 
   {c}
   /. \
  /.   \

node_of_a.children["T"] = node_of_t
node_of_c = TrieNode()

    A Trie class will have all the operations defined in it The most important thing it needs to keep track of is the root for which we will start any operation

    class TrieNode:
	def __init__(self):
		self.children = {}
		self.is_end_of_word = False

"""
trie tree 
   root()
     /
    /
 . {C}
   /. \
  /.   \
 {a}.   {o}

{T}*

node_of_t = TrieNode()

node_of_a = TrieNode()
node_of_a.children["T"] = node_of_t
node_of_c = TrieNode()
node_of_c.children["A"] = node_of_a
node_of_c.children["O"] = node_of_o

Class Trie:
	def __init__(self):
		self.root = TrieNode()
	
	def insert(self, word):
		current = self.root # get the root node
		for char in word: # for each character in the word
			if char not in current.children: # check if the char is present in the children of the root node if not add them
				current.childeren[char] = TrieNode()
			# move the current pointer down the tree 
			current = current.children[char]
				
		current.is_end_of_word = True
	
	def search(self, word):
		current = self.root
		for char in word:
			if char not in current.children:
				return False
			current = current.children[char]
		
		return current.is_end_of_word:

	
	def startwith(self, prefix):
		current = self.root
		for char in prefix:
			if char not in current.children:
				return False
			current = current.children[char]
		
		return True
    Key point to remember :  the root is not a letter; it’s the gatekeeper to all the first letters. The self.root in our Trie is a dummy node. Think of it as the empty starting block before the race begins. It doesn’t represent ‘b’, ‘c’, or any other character. It’s just a container whose children are the actual first letters of all the words we’ve inserted.

    The Coach’s Debrief (Complexity Analysis)

    Let’s quickly analyze this. Let L be the length of the word or prefix we are working with.

    • Time Complexity:

      • insert(word): O(L) - We iterate through the word once.

      • search(word): O(L) - We iterate through the word once.

      • startsWith(prefix): O(L) - We iterate through the prefix once.
        This is incredibly fast and efficient.

    • Space Complexity:

      • In the worst case (no shared prefixes, like “a”, “b”, “c”), it’s O(Total number of characters in all words).

      • In the best case (many shared prefixes, like “apple”, “apply”, “applesauce”), it’s much better than storing the words in a list, because the common prefixes are stored only once.
